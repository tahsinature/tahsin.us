# An Interactive Guide to SVG Paths

SVG gives us many different primitives to work with — `<rect>`, `<circle>`, `<line>`, `<polygon>` — but by far the most powerful is the `<path>` element. It can draw anything the other elements can draw, and a whole lot more.

![Abstract geometric shapes and colorful lines — SVG paths can create any shape imaginable from simple commands.](./svg-paths-hero.jpg)

Unfortunately, the `<path>` element is also the most inscrutable, with its compact regex-style syntax. In this tutorial, let's demystify this infamous element.

---

## The `d` Attribute

Every SVG path is defined by a single attribute: `d` (short for "data"). This attribute contains a series of commands that tell the browser how to draw the path.

Here's a simple example:

```html
<svg viewBox="0 0 200 200">
  <path d="M 10 80 C 40 10, 65 10, 95 80 S 150 150, 180 80" fill="none" stroke="currentColor" stroke-width="2" />
</svg>
```

Each letter in the `d` string is a **command**, and the numbers that follow are its **parameters**.

## Move and Line Commands

The most basic commands are:

- **M** (Move To) — picks up the pen and moves to a new point
- **L** (Line To) — draws a straight line from the current position
- **H** (Horizontal Line) — draws a horizontal line
- **V** (Vertical Line) — draws a vertical line
- **Z** (Close Path) — draws a line back to the start

```html
<svg viewBox="0 0 200 200">
  <!-- A triangle -->
  <path d="M 100 10 L 190 190 L 10 190 Z" fill="none" stroke="#4fc3f7" stroke-width="2" />
</svg>
```

## Bézier Curves

This is where it gets really fun. SVG supports two types of Bézier curves:

### Quadratic Bézier (Q)

A quadratic Bézier curve has one control point. The syntax is `Q cx cy, ex ey` where `(cx, cy)` is the control point and `(ex, ey)` is the endpoint.

![Mathematical curves and wave patterns — the foundation of how Bézier curves work in computer graphics and SVG.](./bezier-curves.jpg)

### Cubic Bézier (C)

A cubic Bézier has **two** control points, giving you much more flexibility over the curve shape:

```html
<svg viewBox="0 0 200 200">
  <path d="M 10 100 C 30 10, 70 10, 100 100 C 130 190, 170 190, 190 100" fill="none" stroke="#ff6b9d" stroke-width="2" />
</svg>
```

The S command (smooth cubic) is a shorthand that automatically mirrors the previous control point, creating smooth connections between curves.

## Arcs

The arc command (`A`) is the most complex path command. It draws an elliptical arc with these parameters:

```
A rx ry x-rotation large-arc-flag sweep-flag x y
```

Here's a practical example — drawing a pie chart segment:

```html
<svg viewBox="0 0 200 200">
  <path d="M 100 100 L 100 10 A 90 90 0 0 1 190 100 Z" fill="#ffd644" opacity="0.8" />
</svg>
```

The two flags (`large-arc-flag` and `sweep-flag`) determine which of the four possible arcs to draw, since any two points on an ellipse can be connected by four different arcs.

## Putting It All Together

Here's a remote image showing what's possible when you combine these commands:

![A decorative abstract design created entirely with SVG path commands](https://images.unsplash.com/photo-1558591710-4b4a1ae0f04d?w=800&q=80)

The real power of SVG paths is that they're fully programmable. You can generate them with JavaScript, animate them with CSS, and manipulate them with the DOM API. They're not just static images — they're living, breathing elements on your page.

> **Pro tip:** The best way to learn SVG paths is to experiment. Open your browser's dev tools, find an SVG path, and start tweaking the numbers. You'll build intuition quickly!
